49     * Registers an identifier that is either brand new, has been reserved or is currently marked as deleted.
50     * It assigns the latest metadata and a URL for resolution.
51     * This causes the DOI to be publicly registered with resolvers and other external services.
52     * <br/>
53     * DataCite and EZID restrict target URL domains and the target URL given MUST have a domain
54     * matching your account permissions.
55     * <br/>
56     * You can also call this method to re-register previously deleted, registered DOIs
57     * that are currently marked as inactive (DataCite) or unavailable (EZID). Because the API of these 2 implementations
58     * differ in how to re-register deleted DOIs (DataCite requires new metadata, EZID needs a new terget URL),
59     * we have to take both the metadata and the target URL as params.
60     *
61     * @param doi the identifier to register
62     * @param target the URL the DOI should resolve to
63     * @param metadata the metadata to be associated with the doi. The identifier inside the metadata will be overwritten
64     *                 by the explicitly given DOI
65     *
66     * @throws DoiException if the operation failed for any reason
67     * @throws DoiExistsException if the DOI was already registered
68     */
69    void register(DOI doi, URI target, DataCiteMetadata metadata) throws DoiExistsException, DoiException;

71    /**

73     * if it was registered before it cannot be deleted as DOIs are permanent identifiers.
74     * In DataCite a DOI will be marked as "inactive" though, in EZID as "unavailable".
75     * You can re-register a deleted DOI again if needed.
